## 🤝 Code of Conduct

All contributors must maintain a professional, respectful environment.

---

## 🔄 Development Workflow

1. **Pull latest main** before starting work
2. **Create a feature branch** from `main`
3. **Make changes** following [CODING_STANDARD.md](./CODING_STANDARD.md)
4. **Write/update tests** for changed code
5. **Run linting and tests** before committing
6. **Open a Pull Request** against `main`
7. **Await code review** from at least one team member
8. **Merge after approval**

---

## 🌿 Branch Strategy

```
main        → Production-ready code
staging     → Pre-release staging
feature/*   → New features
fix/*       → Bug fixes
docs/*      → Documentation updates
chore/*     → Maintenance tasks
```

---

## ✍️ Commit Guidelines

Follow [Conventional Commits](https://conventionalcommits.org):

```
feat(hero): add animated background slider
fix(form): resolve email validation not showing error
docs(api): add contact endpoint examples
```

---

## 🔀 Pull Request Process

1. Fill in the PR template completely
2. Link related issues
3. Add screenshots for UI changes
4. Ensure all CI checks pass
5. Request review from a team member

---

## 👀 Code Review

Reviewers check for:
- Correctness and logic
- TypeScript types
- Accessibility
- Performance implications
- Test coverage
- Documentation
